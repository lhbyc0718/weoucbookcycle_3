## 🛠️ 故障排查

### 常见问题快速定位

**症状：消息未收到**
1. 检查 WebSocket 连接状态 (DevTools -> Network)
2. 检查后端日志中是否有 Redis 发布错误
3. 检查浏览器是否订阅了 'message' 事件
4. 重新刷新页面

**症状：未读数不正确**
1. 检查 activeChat 是否正确设置
2. 检查 Redis key: `unread:userId:chatId`
3. 检查数据库 chat_users 表的 unread_count
4. 清除缓存重试

**症状：聊天列表为空**
1. 检查数据库中是否有 chat_users 记录
2. 检查 Redis 缓存 key: `chat:*`
3. 检查权限是否正确验证
4. 重新登录

**症状：首条消息发送失败**
1. 检查后端创建 chat 是否成功（数据库）
2. 检查返回的 chatId 是否有效
3. 检查网络延迟是否过高
4. 查看后端业务日志
