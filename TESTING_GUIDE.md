# 聊天功能测试指南

本文档提供了完整的功能测试清单和步骤。

## 🎯 测试前检查

### 环境准备
- [ ] 后端服务启动（`go run main.go`）
- [ ] 前端开发服务器启动（`npm run dev`）
- [ ] Redis 服务正常运行
- [ ] MySQL 数据库连接正常
- [ ] 浏览器已打开 `http://localhost:5173`

### 测试账户
- 创建至少 2 个测试账户用于聊天测试

## 📊 功能测试矩阵

### 1️⃣ 图片上传功能测试

#### 1.1 基础上传测试
```
步骤：
1. 打开聊天页面
2. 点击📎(图片)按钮
3. 选择一个 PNG/JPG 图片（< 5MB）
4. 观察上传进度

预期结果：
✅ 上传进度条从 0% 逐渐增加到 100%
✅ 上传完成后显示"图片上传成功"提示
✅ 图片自动显示在聊天消息中
✅ 图片点击可在新标签页打开
✅ 另一个用户能看到这张图片
```

**测试用例**
| 测试项 | 输入 | 预期 | 实际 | 状态 |
|--------|------|------|------|------|
| 小图片上传 | 1MB JPG | 上传成功 | | ☐ |
| 大图片上传 | 4.5MB PNG | 上传成功 | | ☐ |
| 超大图片 | 10MB JPG | 显示"大小超过5MB"错误 | | ☐ |
| 错误格式 | .txt 文件 | 显示"不支持的格式"错误 | | ☐ |
| WebP 格式 | .webp 图片 | 上传成功 | | ☐ |
| GIF 动图 | .gif 文件 | 显示图片（可能不动） | | ☐ |

#### 1.2 网络异常测试
```
步骤：
1. 打开浏览器开发者工具 (F12)
2. 打开 Network 标签
3. 选择"Offline"中断网络
4. 尝试上传图片
5. 恢复网络连接

预期结果：
✅ 显示上传失败提示
✅ 错误信息清晰提示网络问题
✅ 可重新选择文件重试
```

#### 1.3 并发上传测试
```
步骤：
1. 快速点击图片按钮多次选择不同图片
2. 或拖拽多张图片到聊天区域（如果支持）

预期结果：
✅ 每张图片单独上传
✅ 进度条正确显示
✅ 所有图片都在聊天中显示
```

## 🔍 调试技巧

### 浏览器开发者工具检查

**Network 标签**
- 监控 `/api/upload` 请求
- 检查上传响应格式
- 验证状态码（200 成功）

**Console 标签**
- 检查 JavaScript 错误
- 验证 API 响应日志
- 检查 WebSocket 连接状态

**Application 标签**
- 检查 LocalStorage 中的聊天记录
- 验证 WebSocket 消息

### 后端日志检查

```bash
# 查看上传相关日志
grep -i "upload" go.log

# 查看表情API日志
grep -i "emoji" go.log

# 查看错误日志
grep -i "error" go.log
```

### 数据库验证

```sql
-- 检查聊天消息记录
SELECT * FROM messages 
WHERE type IN ('image', 'text') 
ORDER BY created_at DESC LIMIT 10;

-- 检查特定用户的消息
SELECT * FROM messages 
WHERE sender_id = 'user_id' 
ORDER BY created_at DESC;
```
